# Conventions & règles de collaboration

Cette section regroupe les deux cadres de référence qui gouvernent nos échanges et le code du projet. Garde ces documents sous la main lorsque tu me sollicites : ils servent de contrat partagé et sont mis à jour dès qu'une nouvelle règle est décidée.

- [`agent-guidelines.md`](./agent-guidelines.md) — Règles d'intervention Codex : validation explicite, commandes interdites, langue de réponse, gestion du journal partagé…
- [`project-guidelines.md`](./project-guidelines.md) — Lignes directrices techniques : architecture, typage, i18n, DTOs, UI, observabilité.

> ℹ️ Avant toute nouvelle session, vérifie rapidement si ces fichiers ont évolué. Quand une décision transversale est prise pendant l'analyse d'un fichier, on documente la règle ici avant de poursuivre la rédaction.
